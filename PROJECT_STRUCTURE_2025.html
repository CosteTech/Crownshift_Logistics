Core Architecture

The application follows a modern Next.js App Router structure with clear separation of concerns:
  • src/app - Next.js App Router structure with pages, API routes, and components
  • src/components - Reusable React components for UI and auth flows
  • src/lib - Utility functions, Firebase initialization, and business logic
  • src/firebase - Firebase client provider and authentication setup
  • src/hooks - Custom React hooks for auth, mobile detection, and toasts
  • middleware.ts - Edge-safe route protection and session validation
  • scripts/ - Node.js-only utilities (invoice generation, serial numbers)

Session Management Architecture

The application implements server-side session management to enable protected routes on Edge runtime:
  • Firebase ID tokens converted to HTTP-only secure cookies via /api/auth/session
  • 7-day cookie expiration with production-only secure flag
  • Middleware checks cookie presence, redirects unauthenticated users to login
  • Login handlers: Get ID token → POST to session endpoint → Redirect on success
  • Logout action: Deletes session cookie via DELETE request to session endpoint
  • Zero parsing in Edge runtime - only presence check for performance
